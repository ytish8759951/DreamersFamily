# DreamersFamily Full-Site Cross-Device Sync Matrix

Last audited: 2026-07-23 Asia/Taipei

## Product Rule

Any data that a parent or child has explicitly saved/submitted is formal business data and must use Supabase as the source of truth. IndexedDB and localStorage are allowed only for unsent drafts, media previews, offline cache, session/device binding, and UI state.

## Root Causes Found

1. `task_records`, `badges`, and `child_badges` were not fully represented in the parent Realtime/fetch path, so some cross-device updates depended on unrelated table changes or stale repository snapshots.
2. Several shared features still use a cache-first `delegateWrite()` pathway. This is not by itself wrong for offline support, but formal business operations that change balances or states must be promoted to scoped RPCs with idempotency.
3. Media flows are mostly formalized for shares and existing media repositories, but old IndexedDB-only media can only be repaired from the original device that still has the Blob.
